# Chrome 浏览器集成

Chrome 工具需通过 `claude --chrome` 启动。仅在以下阶段允许使用。

## 允许的阶段

| 阶段 | Chrome | 用途 |
|------|--------|------|
| P2 | **允许** | UI/UX 调研、竞品分析、**原型预览** |
| P4 | **允许** | 视觉回归测试、E2E 交互验证、响应式测试 |
| 其他 | 禁止 | — |

## P2 使用规范

### UI 设计调研
- 访问竞品和 UI 组件库文档
- 截图保存设计参考
- 禁止登录私人账户、提交表单、下载文件

### 原型展示（核心流程）
涉及 UI 的任务必须在 P2 创建原型并在 Chrome 中展示给用户确认：

1. **创建原型 HTML** — 写一个自包含的 HTML 文件（内联 CSS + JS），展示页面布局、配色、组件样式、交互效果
2. **Chrome 打开预览** — 用 Chrome 打开本地 HTML 文件，让用户直观看到设计效果
3. **截图记录** — 截取关键页面截图作为设计参考
4. **迭代优化** — 根据用户反馈修改原型，再次预览，直到用户满意
5. **确认后进入 P3** — 用户确认原型设计后，P3 编码以此原型为视觉基准

### 原型文件规范
- 文件路径：项目根目录或 `.claude/prototype/` 下
- 格式：单个 HTML 文件，内联所有 CSS/JS（自包含，无外部依赖）
- 可引用 CDN 上的组件库（如 Tailwind CSS CDN、DaisyUI CDN 等）
- 原型仅用于设计确认，P3 编码时用正式的组件库和构建工具重写

## P4 使用规范
- 打开本地开发服务器验证 UI
- 截图对比渲染结果
- 验证交互流程和响应式布局
- 禁止访问外部网站
